265 /*
266  * Note signing digests and streaming APIs
267  *
268  * Sometimes signaure APIs include "sign_digest" and "verify_digest" functions,
269  * that allow the caller to hash the message themselves and provide the digest
270  * to the sign/verify functions. Example use cases for these APIs are
271  * applications that must sign very large messages, which don't fit in memory,
272  * or that must signa long stream data. Applications that really need this
273  * behavior could provide a digest to the sign/verify APIs provided here.
274  * This is discouraged however, because the message will then be hashed twice,
275  * the challenge used in the proof will be:
276  * c = H(random, per-signature values || H(M))
277  * instead of
278  * c = H(random, per-signature values || M).
279  * In the former case, a collision for H immediately gives a forgery (in the
280  * GMR security game, but not necessarily in all applications). For this reason
281  * the latter case arguably provides better security.
282  *
283  * A better way to implement a streaming API (not provided by this library)
284  * would be to hash the first part of the challenge, then continue hashing the
285  * message as a stream, then finish computing the signature.
286  *
287  */
